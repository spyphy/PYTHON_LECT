\section{First application}

Создаем файл hello.py:

\begin{minted}{python}
from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello World!'

if __name__ == '__main__':
    app.run()
\end{minted}

Запускаем:

\verb|python3 hello.py|

Вс\"e! По адресу \url{http://127.0.0.1:5000/} будет наша страничка.

Созданный объект \verb|app| класса \verb|Flask| --- это и есть наше WSGI-приложение (WSGI --- Web Server Gateway Interface).

\section{Jinja2}

Jinja2 --- язык шаблонов для Flask и не только.


\begin{minted}{jinja}
{% if not session.logged_in %}
	{{ data.html | safe }}
{% else %}
	{{ somedata }}
{% endif %}

{% for entry in entries %}
	<li><h2>{{ entry.title }}</h2>{{ entry.text|safe }}
{% else %}
	<li><em>Unbelievable.  No entries here so far</em>
{% endfor %}
\end{minted}


Разделители (delimiter):
\begin{minted}{jinja}
{% ... %} for Statements
{{ ... }} for Expressions to print to the template output
{# ... #} for Comments not included in the template output
#  ... ## for Line Statements
\end{minted}

Переменные:
\begin{minted}{jinja}
{{ foo.bar }}
{{ foo['bar'] }}
\end{minted}

\section{Flask + Apache}

{\bf WSGI} (англ. Web Server Gateway Interface) --- стандарт взаимодействия между Python-программой, выполняющейся на стороне сервера, и самим веб-сервером, например Apache.

WSGI предоставляет простой и универсальный интерфейс между большинством веб-серверов и веб-приложениями или фреймворками.

По стандарту, WSGI-приложение должно удовлетворять следующим требованиям:
\begin{itemize}
 \item должно быть вызываемым (callable) объектом (обычно это функция или метод)
 \item принимать два параметра:
 \item словарь переменных окружения (environ)
 \item обработчик запроса (\verb|start_response|)
 \item вызывать обработчик запроса с кодом HTTP-ответа и HTTP-заголовками
 \item возвращать итерируемый объект с телом ответа
\end{itemize}


\subsection{Установка mod-wsgi}$~$

\verb|apt-get install libapache2-mod-wsgi|

\subsection{Настройка Apache}

\begin{minted}{html}
<VirtualHost *>
    ServerName example.com

    WSGIDaemonProcess yourapplication user=user1 group=group1 threads=5
    WSGIScriptAlias / /var/www/yourapplication/yourapplication.wsgi

    <Directory /var/www/yourapplication>
        WSGIProcessGroup yourapplication
        WSGIApplicationGroup %{GLOBAL}
        Order deny,allow
        Allow from all
    </Directory>
</VirtualHost>
\end{minted}



\subsection{Создание файла .wsgi}

Этот файл (в корне проекта) содержит код, выполняемый \verb|mod_wsgi| для получения объекта приложения:

\begin{minted}{python}
from yourapplication import app as application
\end{minted}

Объект с именем \verb|application| будет использоваться в качестве приложения.

На диске должен быть также файл \verb|yourapplication.py|, в котором создается объект app класса flask.

\vspace{4mm}

\section{Dash, React}

Dash: \url{https://tproger.ru/translations/reactive-web-apps-with-python/}

React: \url{https://tproger.ru/translations/react-basic-weather-app/}


Приложения на \verb|Dash| --- веб-серверы, которые запускают Flask и связывают пакеты \verb|JSON| через HTTP-запросы. Интерфейс Dash формирует компоненты, используя \verb|React.js|.

Install
\begin{verbatim}
sudo pip3 install dash dash-renderer dash-html-components dash-core-components plotly
\end{verbatim}

Чтобы Dash работал, нужно в wsgi-файле строку
\begin{minted}{python}
from myapp import app as application
\end{minted}
заменить на
\begin{minted}{python}
from myapp import app
application = app.server
\end{minted}
